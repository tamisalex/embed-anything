# Quickstart — Manual Setup Steps

Everything in this project is automated except for the one-time steps below.
Complete them in order before running the pipeline for the first time.

---

## 1. Prerequisites

- AWS account with an IAM user that has admin permissions
- AWS CLI configured (`aws configure`)
- OpenTofu ≥ 1.7 (`brew install opentofu`)
- Docker Desktop running
- Python 3.12+ and `uv` (`brew install uv`)
- A [Prefect Cloud](https://app.prefect.cloud) account (free tier)
- A [Pinecone](https://app.pinecone.io) account (free tier)

---

## 2. Pinecone index

1. Log in to Pinecone → **Create Index**
2. Settings:
   - **Name**: `embed-anything`
   - **Dimensions**: `512`
   - **Metric**: `Cosine`
   - **Type**: `Dense`
   - **Cloud / Region**: AWS `us-east-1`
3. Copy the **index host** (e.g. `embed-anything-xxxxx.svc.aped-4627-b74a.pinecone.io`)
4. Copy your **API key** from the Pinecone dashboard

---

## 3. Store the Pinecone API key in Secrets Manager

```bash
aws secretsmanager create-secret \
  --name pinecone_api_key \
  --secret-string "YOUR_PINECONE_API_KEY" \
  --region us-east-1
```

This is the only secret you manage manually. Everything else is generated by Tofu.

---

## 4. Deploy AWS infrastructure

```bash
cd infra/terraform/environments/dev
tofu init
tofu apply
```

This provisions: VPC, ECR repos, ECS clusters (pipeline + API), S3 bucket,
Secrets Manager secrets, IAM roles for GitHub Actions and Prefect OIDC,
and CloudWatch log groups.

Note the outputs — you'll need them later:
```bash
tofu output
```

---

## 5. GitHub Actions — push image to ECR

Add these secrets to your GitHub repo (**Settings → Secrets → Actions**):

| Secret | Value |
|--------|-------|
| `AWS_ACCOUNT_ID` | your 12-digit AWS account ID |
| `PREFECT_API_KEY` | your Prefect Cloud API key (from step 6) |

The `github_actions_role_arn` from `tofu output` is already referenced in the
workflow via OIDC — no static AWS keys needed.

Push to `main` to trigger the build. Both `embed-pipeline` and `embed-api`
images will be built and pushed to ECR automatically.

---

## 6. Prefect Cloud

1. Create a free account at [app.prefect.cloud](https://app.prefect.cloud)
2. Copy your **API key** from Account Settings → API Keys
3. Log in locally:
   ```bash
   .venv/bin/prefect cloud login -k YOUR_PREFECT_API_KEY
   ```

No blocks or agents needed — the flow runs locally and submits ECS tasks
directly via boto3.

---

## 7. Load sample data to S3

```bash
cd scripts
uv run python load_data.py \
  --bucket embed-anything-input-bucket \
  --dataset food101 \
  --count 500
```

This uploads images to S3 and writes a `manifest.parquet` that the pipeline
reads as its input.

---

## 8. Create the shared Python environment

From the repo root:

```bash
uv venv .venv --python 3.12
uv pip install -e packages/embed-core -e packages/embed-pipeline
```

---

## 9. Run the pipeline

```bash
.venv/bin/python -c "
from embed_pipeline.flow import embed_pipeline_flow

embed_pipeline_flow(
    athena_results_s3_uri='s3://embed-anything-input-bucket/food101-sample/manifest.parquet',
    tracking_s3_location='s3://embed-anything-input-bucket/iceberg',
    tracking_results_bucket='embed-anything-input-bucket',
    pipeline_index='food101',
    pipeline_run_id='food101-run-001',
    ecs_subnet_id='SUBNET_ID',   # from: tofu output pipeline_task_sg_id
)
"
```

Monitor progress in [Prefect Cloud](https://app.prefect.cloud) and
CloudWatch Logs (`/ecs/ea-dev-pipeline`).

---

## What's fully automated (no manual steps)

| What | How |
|------|-----|
| Docker image builds | GitHub Actions on push to `main` |
| AWS infrastructure | `tofu apply` |
| Pinecone API key injection | ECS reads from Secrets Manager at container start |
| Athena Iceberg tracking tables | Created by pipeline on first run |
| Pinecone index creation | Created by store on first `initialize()` if not present |
| CloudWatch logging | Configured in ECS task definition by Tofu |
